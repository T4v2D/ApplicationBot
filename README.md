# Telegram‑бот для приёма заявок

Простой Telegram‑бот на базе библиотеки `aiogram`, позволяющий пользователям оставлять заявки, а администратору – просматривать их и контролировать количество.

## Возможности

- **Для обычных пользователей:**
  - Подача заявки (имя, контакты, комментарий).
  - Получение подсказки по команде «Помощь».
- **Для администратора:**
  - Просмотр последней заявки.
  - Подсчёт общего количества заявок.
  - Автоматическое уведомление о каждой новой заявке.

> ⚠️ **Важно:** первый пользователь, запустивший бота, автоматически становится администратором. Это реализовано для упрощения демонстрации.

## Технологии

- Python 3.7+
- [aiogram](https://docs.aiogram.dev/) – асинхронный фреймворк для Telegram Bot API.
- SQLite – встроенная база данных для хранения заявок и идентификатора администратора.

## Установка и запуск

1. **Клонируйте репозиторий**
   ```bash
   git clone https://github.com/yourusername/your-repo.git
   cd your-repo
   ```

2. **Установите зависимости**
   ```bash
   pip install aiogram
   ```
   (SQLite встроен в Python, дополнительных установок не требуется.)

3. **Получите токен бота**
   - Напишите [@BotFather](https://t.me/botfather) в Telegram, создайте бота и скопируйте токен.

4. **Вставьте токен в код**
   - Откройте файл с ботом (например, `bot.py`).
   - В строке `TOKEN = ""` вставьте свой токен между кавычек.

5. **Запустите бота**
   ```bash
   python bot.py
   ```

## Использование

Бот взаимодействует с пользователями через кнопки и текстовые сообщения.

### Команды

- `/start` – приветственное сообщение. Определяет, является ли пользователь администратором, и показывает соответствующие кнопки.

### Кнопки (для обычных пользователей)

| Кнопка | Действие |
|--------|----------|
| **Оставить заявку** | Запускает последовательный опрос: имя → контакты → комментарий. После завершения заявка сохраняется в БД, а администратор получает уведомление. |
| **Помощь** | Выводит краткую подсказку. |

### Кнопки (для администратора)

| Кнопка | Действие |
|--------|----------|
| **Просмотреть последнюю заявку** | Показывает данные последней оформленной заявки. |
| **Количество заявок** | Выводит общее число заявок в базе. |

## Структура базы данных

- Таблица `Applications`:
  - `ID` – автоинкрементный первичный ключ.
  - `UserID` – Telegram ID пользователя, оставившего заявку.
  - `DateTime` – дата и время подачи.
  - `Name` – имя, указанное пользователем.
  - `Contacts` – контактная информация.
  - `Comments` – комментарий пользователя.

- Таблица `Admin`:
  - `UserId` – Telegram ID администратора (хранится одна запись).

## Важные замечания

- **Администратор** назначается **первым**, кто отправит команду `/start`. Если администратор уже существует, другие пользователи не получат прав.
- **Токен бота** хранится прямо в коде – для продакшена рекомендуется использовать переменные окружения или отдельный конфигурационный файл.
- **Обработка ошибок** ограничена – например, при отправке уведомления администратору используется `try/except`, но другие потенциальные ошибки (например, закрытие курсора после коммита) могут привести к непредсказуемому поведению.
- База данных `applications.db` создаётся автоматически в той же папке, где запущен скрипт.
