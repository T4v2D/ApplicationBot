# Telegram-бот для приёма заявок

Данный проект представляет собой телеграм-бота, разработанного для автоматизации сбора заявок от пользователей. Бот позволяет клиентам оставлять заявки с указанием имени, контактных данных и комментария, а администратору – просматривать последнюю заявку и общее количество заявок. Все данные сохраняются в локальной базе данных SQLite.

Проект написан на **Python** с использованием библиотеки **aiogram** (асинхронный фреймворк для Telegram Bot API) и встроенного модуля **sqlite3**. Реализован конечный автомат (FSM) для пошагового сбора информации от пользователя.

---

## Основные возможности

### Для обычного пользователя:
- **Оставить заявку** – последовательный ввод имени, контактных данных (телефон, email и т.п.) и комментария. После сохранения заявки пользователь получает подтверждение.
- **Помощь** – краткая подсказка о том, как воспользоваться ботом.

### Для администратора:
- **Просмотреть последнюю заявку** – вывод полной информации о самой свежей заявке (ID, ID пользователя, время, имя, контакты, комментарий).
- **Количество заявок** – показывает общее число сохранённых заявок.

> **Важно:** первый пользователь, запустивший бота, автоматически становится администратором. В дальнейшем администратор может управлять ботом через специальные кнопки.

---

## Технологии

- **Python 3.9+**
- **aiogram 3.x** – асинхронная работа с Telegram API
- **SQLite3** – лёгкая встроенная СУБД для хранения заявок
- **asyncio** – асинхронное выполнение
- **FSM (Finite State Machine)** – управление состояниями диалога

## Установка и запуск

1. **Клонируйте репозиторий:**
   ```bash
   git clone <url-репозитория>
   cd <папка-проекта>
   ```

2. **Создайте виртуальное окружение (рекомендуется):**
   ```bash
   python -m venv venv
   source venv/bin/activate  # для Linux/MacOS
   # или
   venv\Scripts\activate     # для Windows
   ```

3. **Установите зависимости:**
   ```bash
   pip install aiogram
   ```
   (другие библиотеки входят в стандартную поставку Python)

4. **Получите токен бота** у [@BotFather](https://t.me/botfather) в Telegram и вставьте его в переменную `TOKEN` в файле `main.py`.

5. **Запустите бота:**
   ```bash
   python main.py
   ```

## Структура базы данных

Используется одна база данных `applications.db` с двумя таблицами:

### Таблица `Applications`
| Поле        | Тип        | Описание                           |
|-------------|------------|------------------------------------|
| ID          | INTEGER    | Первичный ключ, автоинкремент      |
| UserID      | INTEGER    | Telegram ID пользователя           |
| DateTime    | TEXT       | Дата и время создания заявки       |
| Name        | TEXT       | Имя пользователя                   |
| Contacts    | TEXT       | Контактная информация              |
| Comments    | TEXT       | Комментарий к заявке               |

### Таблица `Admin`
| Поле   | Тип     | Описание                       |
|--------|---------|--------------------------------|
| UserId | INTEGER | Telegram ID администратора (один) |

## Планируемые улучшения

Проект находится в стадии разработки, и в будущем планируется:

- **Исправление отображения данных** – корректный вывод ID заявки и контактов в уведомлении администратора (на данный момент допущена опечатка в синтаксисе f-строк).
- **Многопользовательский доступ** – возможность назначения нескольких администраторов.
- **Удаление заявок** – функция для администратора по удалению обработанных заявок.
- **Экспорт в Excel/CSV** – выгрузка всех заявок в структурированный файл.
- **Обработка ошибок** – более детальное логирование и устойчивость к сбоям.
- **Настройка через переменные окружения** – вынос токена и параметров БД в `.env`.
